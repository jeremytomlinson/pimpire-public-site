Git Quick Reference

  • git remote show origin
    If the remote branch you want to checkout is under "Remote branches" and not "Local Refs" then you need to fetch them first.
  • git fetch
    This will synchronize you with the repo, downloading any data that you do not have locally and giving you bookmarks to where each branch on that remote was when you synchronized.
  • git fetch origin remote-name
    Downloads and syncs you with a specific branch of the repo.
  • git checkout -b local-name origin/remote-name
    git doesn't allow you to work on someone else's branches. You can only work on your own. So if you want to add to someone else's branch, you need to create your own "copy" of that branch, which is what the above command does (it creates your local branch and checks it out, too).
  • git checkout local-name
    Switches to a specific local branch.
  • git pull origin remote-name
    This command will basically run a git fetch immediately followed by a git merge of the branch on that remote that is tracked by whatever branch you are currently in.
  • git branch local-name
    Creates a new local branch based on the branch you're on.
  • git branch -d local-name
    Deletes a local branch.
  • git merge local-branch-name
    Merges a branch context into your current one.
